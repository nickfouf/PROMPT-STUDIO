I've updated the file watcher and the UI to support the new features. Here is what has been changed:

1. **Remove Folders from Project**:
   - Added a "Trash" icon specifically next to the "Root" badge in the File Tree Sidebar.
   - When clicked, it removes that root folder directly from the project's internal nodes list.

2. **Efficient File Watcher (Fixing the full reload issue)**:
   - Previously, the main process was aggressively recreating the `chokidar.watch` instance whenever the list of folders was updated. This wiped out its internal index and re-scanned everything (which triggered the loading spinner and delayed interaction).
   - I updated `main/index.js` (and the `out/` counterpart) to maintain an active tracking set (`watchedPaths`). Now, `currentWatcher.add()` and `currentWatcher.unwatch()` are used to simply patch the differences under the hood while leaving the existing scan intact.
   - Updated the `useFileWatcher` React hook to avoid blindly firing `setIsWatcherReady(false)` unless you are completely booting up the environment initially (or switching to an entirely different project). This makes adding folders *completely seamless* without reloading delays or blocking UI.
   - Added `key={activeProject.id}` directly on `ProjectWorkspace` in `App.jsx` to correctly unmount hooks and cleanly reset states when switching between completely different projects.
