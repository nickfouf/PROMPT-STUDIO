I understand what's happening. You are encountering two compounding issues that cause the "Project files" and "Tokens" to hang infinitely:

1.  **`isCountingTokens` getting stuck in the save file**: Because the UI automatically syncs the `projects` state to `config.json` every 500ms, if the app happens to save right while tokens are being counted, it saves `"isCountingTokens": true`. When you reload the app, it boots up with the token spinner instantly turned on, and gets permanently stuck there if the workspace isn't fully ready yet.
2.  **Infinite Hanging in the Backend (Tokens & Chokidar)**:
      - `chokidar`'s `.close()` method is notoriously known to hang indefinitely on Windows if there are pending active file events. If you add a folder, it tries to close the old watcher and gets entirely stuck, leaving the folder permanently displaying the `_isScanning` spinner.
      - The token counting library (`@lenml/tokenizer-gemini`) internally tries to fetch the 3MB `tokenizer.json` model from Hugging Face on its first run. If your network blocks it or is slow, the native Node `fetch` has no default timeout, permanently hanging the token counting process.

I have applied a patch to strip ephemeral loading flags from the configuration file on save, and wrapped both the Tokenizer fetch logic and the Chokidar closing functions in `Promise.race` timeouts to guarantee the UI is never permanently blocked by the backend.
